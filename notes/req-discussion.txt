  What we're building: A localhost-only digital circuit design editor — a

  - Multi-select and copy/paste are deferred

  one-shot). A component is placed either by dragging from the palette or by
  clicking the palette then clicking the canvas — either way, placement is
  one-shot and returns to select mode.
  
  Components: Rectangular IC outlines with labeled pins. Pin positions and bus
  groupings are specified in the MD file. Components can be rotated in 90°
  increments; all text (pin labels, U-number, type name) stays upright
  regardless of rotation. U-numbers are auto-assigned sequentially. The palette
  is a flat grid of tiles, one per MD file found at startup.

  Wires: Thin black lines. Click a source pin, click a destination pin — wire
  appears as a straight line. Returns to select tool. In select mode, clicking a
  wire segment adds a bend point at the nearest grid point, which can then be
  dragged to rubber-band the wire. A wire with one dangling endpoint is allowed;
  a wire with no connected endpoints is removed.

  Buses: Thick blue lines with a slash-and-digit width annotation. Width set by
  right-clicking. Dragging a bus onto a component can snap-connect all pins in a
  declared group. Buses follow the same bend-point routing model as wires.

  (position, rotation, U-number, type data copied in with optional per-instance
  overrides), wire routes (including bend points), and bus routes.
  
  Server API: Save design, load design, file navigation (list directory
  contents). Designed to be extensible. Component MD files are loaded at server
  startup.
  
  File storage: Platform-standard app data directory by default; user can choose
